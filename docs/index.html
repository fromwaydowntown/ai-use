// how it works

From save to merged, tracked.

Three steps. Installs in your repo. Runs on your existing git push credentials.

01

IDE hook captures AI edits

When you save a file in Cursor, Claude Code, or Codex, the IDE hook fires. It normalises each line, hashes it with SHA-256, and appends the record locally. No raw code — ever.

tool: claude_code · 47 line-hashes appended
02

Pre-push uploads your ref

On git push, the pre-push hook stores your events as a git blob and force-pushes it to a per-developer ref. No extra tokens, no secrets — just your normal git auth.

refs/ai-attribution/<sha256(email)[:16]>
03

GitHub Action posts the check

On every PR open or sync, the workflow fetches all developer refs, downloads the diff, matches hashes, and posts a Check Run with the AI%. Every developer's contribution, counted.

// features

Built for teams who ship.

Everything you need to understand AI adoption across your engineering org. Nothing you don't.

⚡

Per-PR Check Runs

Every pull request gets a GitHub Check with the AI percentage. Reviewers see it right in the PR — no dashboard to open.

📊

Auto-generated dashboard

A docs/AI_USAGE.md with Mermaid charts refreshes on every merge to main. Per-tool breakdown, weekly trend, KPI cards.

🔒

Zero code exposure

SHA-256 hashes only. No raw lines, no file contents, no prompts. If the repo burns down, no source code is recoverable from ai-use data.

🛠

Cursor, Claude Code, Codex

IDE hooks for Cursor and Claude Code. Codex Desktop is covered by the pre-commit git hook. New tools are one adapter away.

🌐

Multi-developer, no conflicts

Each developer pushes to their own git ref derived from their email hash. Two people pushing at once never collide.

📉

Systematically conservative

Edited lines break their hash. Lines under 8 chars are never attributed. Cross-file moves don't count. We'd rather under-report than falsely accuse.

// privacy by design

Your code never leaves your repo.

Built around one constraint: no raw code exits the repository. Ever.

  • Only SHA-256 hashes of individual lines are stored
  • No prompts, no file contents, no context windows recorded
  • All data lives inside your existing GitHub repo
  • No external SaaS, no tokens beyond your git credentials
  • Works on private repos — everything stays inside
  • Fully open source — read every line of what runs
